ESP32-S3 APC UPS (USB HID) — Architecture & Progress Summary
Date: 2026-03-05
ESP-IDF: v5.3.1-dirty
Project: esp32-s3 Nut node (app-template)
Board: ESP32-S3 (QFN56) with OTG power mod (VBUS supplied via second USB port; OTG port powered via solder mod)
Target UPS: APC UPS VID:PID 051d:0002

Current Version
- main.c CURRENT VERSION: v7

What’s Working (confirmed on hardware)
Milestone 2A.1 (UPS_USB_M2A1)
- Detects APC UPS 051d:0002
- Claims HID interface 0
- Reads interrupt IN endpoint 0x81 (MPS=8)
- Successfully requests/dumps HID Report Descriptor (1049 bytes)
- No “waiting for UPS attach” polling spam

Milestone 3 (UPS_USB_M3) — HID report monitor (change-only)
- USB host installed and client registered successfully
- Device enumeration triggers NEW_DEV and opens device by address
- Parses active config descriptor to find:
  - HID interface number
  - Interrupt IN EP address + MPS
  - HID report descriptor length (from HID descriptor 0x21)
- Claims interface, starts interrupt IN streaming
- Logs HID reports only when the report changes (no spam)

Milestone 4 — Early decode
- Decodes selected report IDs from interrupt IN:
  - 0x0C: battery percent + runtime seconds (little-endian)
  - 0x13: AC present (boolean-ish)
  - 0x16: flags (little-endian 32-bit)
- Example decoded values observed:
  - batt=98%
  - runtime=35280 s
  - AC_present=1
  - flags=0x0000000D

Known Issue / Note
- Control GET_DESCRIPTOR(report) attempted in v7 can fail with ESP_ERR_INVALID_ARG in some builds/tests.
- Not blocking Milestone 5 because:
  - Report descriptor length is already parsed from HID descriptor (0x21)
  - Streaming + decode works from interrupt IN reports
- Keep descriptor dump as optional (best-effort) or remove it until needed.

Milestone 5 — Next planned work (ready to start)
Goal: Convert decoded HID reports into a stable UPS state model and prepare “NUT-style” variables.

Planned Implementation Tasks (Milestone 5)
1) UPS state model
   - Define ups_state_t:
     - battery_percent (uint8_t)
     - runtime_seconds (uint32_t)
     - ac_present (bool)
     - status_flags (uint32_t)
     - last_update_ticks / timestamp (optional)
   - Maintain a single global/current state instance.

2) Update pipeline
   - In intr_in_cb(), after decode:
     - Update fields in ups_state_t when a matching report ID arrives
     - Track which fields have been updated since boot (bitmask)
     - Track “changed” flags so the rest of the system can react.

3) Logging improvements
   - Keep raw HID change-only log (already present)
   - Add structured log when any ups_state field changes:
     - batt/runtime/ac/flags + derived status strings

4) Readout interface (later milestones)
   - Provide a local read API (HTTP/JSON or simple TCP) to expose:
     - battery.charge, battery.runtime, ups.status, input.voltage/present (when available)
   - This bridges to NUT concepts without implementing full NUT server yet.

Immediate Next Step Recommendation
- Proceed with Milestone 5 using the working interrupt IN decode path.
- Treat report descriptor dump as optional/non-critical until we need more fields or dynamic report parsing.
