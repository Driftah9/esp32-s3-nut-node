ChatGPT Communication Preferences (Project Workflows)
======================================================

These preferences describe how I (the user) like to work with ChatGPT across *any* technical project conversation.

1) Default Artifacts & Delivery
- Prefer downloadable files over copy/paste in chat whenever possible (especially for full source files like main.c).
- Only paste small snippets in chat if absolutely necessary (e.g., tiny diffs or a few key lines).
- When providing code intended to be used, provide it as a drop-in file with a clear filename and version tag.

2) Source-of-Truth Documentation
- Maintain a single “Master Engineering Log” / summary document as the source of truth.
- Update the log whenever a milestone/step is confirmed working.
- Each update should include:
  - What changed
  - What is confirmed working
  - Known issues (append-only / status updates)
  - Validation/test results
  - Next steps (clear milestone target)
  - The full verified working code (e.g., main.c) appended at the end

3) Versioning & Structure Expectations
- main.c must keep:
  - A clear version header (CURRENT VERSION)
  - A “Version tracking” block
  - A “REVERT INDEX” block for critical historical gotchas/decisions
  - A “NEXT STEPS” section at the end
- Prefer incremental, regression-safe changes that preserve known-working plumbing.

4) Testing & Reporting Style
- Provide concise, actionable validation checklists (MUST/SHOULD/NICE is ideal).
- When I share logs, I prefer small slices that prove:
  - Boot/version banner
  - Attach/enumerate success
  - Key runtime behavior (a few representative lines)
  - Detach/reattach (if relevant)
- I value thoroughness, reproducibility, and traceability over speed.

5) Conversation Style
- Keep instructions clear and repeatable so any engineer can follow them.
- Avoid endless scrolling walls of text; use artifacts (download files) and short summaries.
- Don’t do background/async promises—deliver results in the current reply.

6) Continuation / New Conversation Injection
- When starting a new conversation, I will provide:
  - The current Master Engineering Log (latest)
  - The current verified working code
  - A short log snippet from the latest run (optional)
- The assistant should immediately align to the declared baseline and next milestone goal.
